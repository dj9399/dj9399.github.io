pokemon Go fq+gps模拟

发表于 2016-07-28   |  

按照网上的教程搞定：

1、把坐标复制到模拟位置中的Google play服务的模拟中
2、这时再打开谷歌地图就已经定位到之前选的位置了。
3、完成之后同时打开谷歌地图和游戏，进入游戏你会发现你的位置就是之前定的位置，不同时打开的话会提示红底gps。

需要注意的是你开的VPN所在地区，一定要和模拟位置一致。比如我用的香港VPN就一定要把位置修改到香港。否则的话会被锁

比较麻烦的一点是，只能通过修改模拟位置来移动，因为GPS被修改了，不过配合上https://pokevision.com 这个神器。

搜索了一下：因为是模拟定位，你需要到gps信号强的位置去再一次走动。只能周末去户外尝试一下。

美国末日

发表于 2016-07-18   |  

​ 睡觉前MP3突然转到了linkin park的”untraveled road”，一下又想起了一前多前看过的纯黑《美国末日：重制版》（”untraveled road”是片尾MV背景音乐,简直绝配）
​ 初看纯黑版本美国末日第一集，感觉就片头片尾MV做得很棒，故事一般般，父亲在疫情爆发之季不幸丧失了自己的女儿…但是冲着还没看到片头中的僵尸，我耐心的看了下去，却看到了乔尔和艾莉之间各种美妙的小故事，失落世界的绿色美丽，少数生存者为了满足生存与兽欲作出的连禽兽不如的行为。
​ 美国末日真不愧是得分最高的游戏，剧情、音乐、画面几乎完美，看完之后感概良多，人类自诩为地球统治者，然而在疫情和天灾面前却经常无能为力。事实上人类制造的武器早已经足以消灭自己，根本无需它族动手。伤害人们最多的是苛政、自身永无何止的欲望、其它人，真正的末日，不是孢子变异，不是没有人，而是人性缺席


​ 感概说得太多，其实这个游戏给人最深的感觉还是乔尔和艾莉之间的父女之情，直到让你想哭。严重推荐没看到此系列视频的朋友去拜读一下，绝对不下于任何一部大片。大概是720p画面成为主流后，（当然，暴雪属于例外），3D的游戏画面已经达到了现实级的水准，出现了很多大作，剧情画面已经不下于好莱坞大片，如果要给此片一个IMDB评分，个人觉得9.0非常合适。

hexo写文章

发表于 2016-07-15   |  

准备-

关于文章目录

  1. scaffolds：这里放着3个markdown文件，分别对应草稿，页面，文章三种类型的页面模板，之后你使用Hexo new命令来新建一个markdown资源文件时，就可以指定使用哪个作为新建文件的模板。比如Hexo new post mynewpost，就会在资源文件夹Post分文件夹新建一个名为mynewpost.md的markdown文件。打开这个文件，写你的文章，之后Hexo博客引擎就能根据这个来直接生成静态的html页面并自动Push到你设置的服务器（或page静态服务的代码库）里。
  2. source：资源文件夹，存放所有的文章的原始markdown文件。markdown是程序员世界最流行的撰写文章的工具，它是一套语言规范，可以使你像码代码一些快速码出带格式的文章。

默认方式写博客

  1. 按照Scaffolds目录下提供的layout之一新建一个文件，当然你也可以写一个属于自己的layout放到此目录下，而写文章要用 post 类型的 layout，而 post 在 Hexo 的配置文件里被配置成了默认的 layout，所以如下命令的 loyout 可省略。

  2. 然后用你喜欢的Markdown编辑器打开刚刚生成的博客文件，写下你想写的文字。

  3. 生成博客：

注意：hexo deploy 时每次都会覆盖原来的内容，所以记得备份。

发布成功后，就可通过 username.github.io来访问你的博客。

用编辑器写博客

新建文章

新手推荐使用文章编辑器进行写作，我是使用了Typora。

完成后，在blog/source/_posts目录下新建一个以.md为后缀的文件，如test.md，然后用Notepad++打开该文件，并把之前写好的内容复制进去保存。
文章的永久链接格式title属性取自该文件名。如使用默认的永久链接格式:year/:month/:day/:title/，则生成的链接为http://yoursite.com/2015/03/21/test/。

预览

在blog目录下右键点击Git Bash Here，执行hexo s启动服务器。

在浏览器中打开http://localhost:4000，即可看到刚才写的文章。
启动服务器后，会实时监控文件，再次修改.md文件只需刷新浏览器即可看到新的效果。

生成静态文件

预览满意后，即可执行hexo g命令生成静态文件。生成的静态文件位于blog/public目录下。

为了避免遇到莫名其妙的问题，建议使用hexo clean && hexo g命令，先清除缓存文件 (db.json) 和已生成的静态文件 (public)，再生成静态文件。

附录：HEXO指令

指令

init
$ hexo init [folder]

新建一个网站。如果没有设置 folder ，Hexo 默认在目前的文件夹建立网站。
new$ hexo new [layout]

新建一篇文章。如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来。
generate$ hexo generate

生成静态文件。
选项描述-d, –deploy文件生成后立即部署网站-w, –watch监视文件变动publish$ hexo publish [layout]

发表草稿。
server$ hexo server

启动服务器。默认情况下，访问网址为： http://localhost:4000/。
选项描述-p, –port重设端口-s, –static只使用静态文件-l, –log启动日记记录，使用覆盖记录格式deploy$ hexo deploy

部署网站。
参数描述-g, –generate部署之前预先生成静态文件render$ hexo render [file2] …

渲染文件。
参数描述-o, –output设置输出路径migrate$ hexo migrate

从其他博客系统 迁移内容。
clean$ hexo clean

清除缓存文件 (db.json) 和已生成的静态文件 (public)。
list$ hexo list

列出网站资料。
version$ hexo version

显示 Hexo 版本。
选项安全模式$ hexo –safe

在安全模式下，不会载入插件和脚本。当您在安装新插件遭遇问题时，可以尝试以安全模式重新执行。
调试模式$ hexo –debug

在终端中显示调试信息并记录到 debug.log。当您碰到问题时，可以尝试用调试模式重新执行一次，并 提交调试信息到 GitHub。
简洁模式$ hexo –silent

隐藏终端信息。
自定义配置文件的路径$ hexo –config custom.yml

自定义配置文件的路径，执行后将不再使用 _config.yml。
显示草稿$ hexo –draft

显示 source/_drafts 文件夹中的草稿文章。
自定义 CWD$ hexo –cwd /path/to/cwd

自定义当前工作目录（Current working directory）的路径。
