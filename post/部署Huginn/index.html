部署Huginn

发表于 2018-03-13

背景

自从收到KINDLE，就发现在这个RSS日益减少的时代，用RSS有多少困难。好不容易找到了KINDLEEAR完美推送RSS，之后又发现有几个常看的RSS（RSS地址打不开却可以在INOREADER中订阅）无法在KINDLEEAR中推送，作为坚持阅读RSS十多年我实在是不能忍。于是乎又找到个神器Huginn，可以完美将任何网站转为RSS订阅。于是再次部署

Huginn 简介

Huginn 是一个用于构建在线执行自动化任务的系统，它可以监控网页变化、聚合第三方应用并且根据设定的触发条件执行自动化操作，你可以将它看作是一个可以运行在自己服务器上的 Yahoo!Pipes + IFTTT。该工具的作者为 Andrew Cantino，Github 上的项目地址为 https://github.com/cantino/huginn/， 已经有上万的 star，而且该项目活跃度非常高，一直都有人在更新代码。
嗯，就抄了上面一点，总之是个神器，而我只使用其1%聚合RSS的功能

安装篇

在公司找了台闲置的老CentOS6.5 64位的机器，准备用docer安装（因为这种方式安装最简单），推荐找个CentOS7 64位的VPS主机。

CentOS6.5 升级内核

centos初始内核为2.6 ，用命令uname -r

  • 在yum的ELRepo源中，有 m ain l ine（3.13.1）、 l ong- t erm（3.10.28）这2个内核版本，考虑到long-term更稳定，会长期更新，所以选择这个版本。
  1. 导入public key :
    rpm --import https://www.elrepo.org/RPM-GPG-KEY-elrepo.org
    解决curl: (35) SSL connect error的解决方法:yum update nss
  2. 安装ELRepo到CentOS-6.5中:
    rpm -Uvh http://www.elrepo.org/elrepo-release-6-8.el6.elrepo.noarch.rpm
  3. 安装 kernel-lt（lt=long-term）:
    yum --enablerepo=elrepo-kernel install kernel-lt -y
    或者安装kernel-ml（ml=mainline）:
    yum --enablerepo=elrepo-kernel install kernel-ml -y
  4. 编辑grub.conf文件，修改Grub引导顺序，确认刚安装好的内核在哪个位置，然后设置default值（从0开始），一般新安装的内核在第一个位置，所以设置default=0。
    vi /etc/grub.conf
  1. 重启
    init 6
  2. 重新查看内核
    uname -r
  1. 大功告成O(∩_∩)O~

CentOS6.5 安装Docker

  1. 更新yum(大概30分钟，看网速)
    yum update
  2. 生成docker.repo

tee 读取标准输入的数据，并将其内容输出成文件。

  1. yum安装docker

docker安装

  1. 查看版本
  1. 启动docker服务
    service docker start
  2. 检查docker状态
    service docker status
  3. 在线查找镜像:docker search [镜像名]
    docker search tomcat
    start代表受欢迎程度。OFFICIAL代表官方版本

拉取镜像：docker pull [镜像名称[:版本]]
docker pull tomcat:7

docker 上运行 Huginn

  1. 使用这个命令启动你的 Huginn 容器 docker run -it -p 3000:3000 cantino/huginn 初次下载容器大概1小时-2小时
  2. 在浏览器里面打开 Huginn http://localhost:3000
  3. 使用默认帐号密码登录 Huginn 网站(admin/password)

重启VPS后，启动Huginn

由于VPS重启后，并没有自动加载镜像，因而重启VPS后，你的Huginn将无法启动，当然启动你的Huginn的方法也非常简单。

1.首先查出你的Huginn容器的id

docker ps -a

2.启动容器

docker start 容器_id/容器_name
