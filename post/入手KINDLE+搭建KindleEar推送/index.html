入手KINDLE+搭建KindleEar推送

发表于 2018-03-05

注意：一定要在FQ姿态进入，没一定的IT基础连FQ也不会的还是不用试了，可以找朋友帮忙弄。

入手KINDLE

每天显示器看得眼睛难受，因为有很多时间都花在读RSS上，所以考虑了很久，终于决定入了个二手KINDLE入门版本来试试水。普通ANDROID电子书没查出来有支持INOREADER的，综合了很多考虑还是选择了KINDLE

本着买新不买旧的原则，在闲鱼看到个最便宜的KINDLE8 300块，卖家信用也不错，带皮套贴膜，图片看来满新的，于是直接下单。到手后感觉果然是不错的东西，后悔没早几年入KINDLE就好了。买来就是为了读RSS，于是二天内尝试了所有主流的RSS推送及网页推送，最后选择了KindleEar

什么是KindleEar

KindleEar 是一款开源的 Python 程序，由网友 cdhigh 发起，托管在 Github。它可运行在免费的 Google APP Engine 上，把 RSS 生成排版精美的杂志模式的 MOBI 文件，并按照设置定时自动推送至你的 Kindle。如果你有 Python 和前端基础，还可以自定义排版，生成你需要的最完美的 MOBI 文件。–引用自书伴

搭建好的KindleEar运行效果

KindleEar实现功能

  1. 定期自动推送。就是定期把一些新闻推送到你的Kindle，完全不用人工干预。
  2. 自动生成图文的mobi或者epub格式。在Kindle上的排版和格式非常好。
  3. 支持RSS订阅。订阅源一般都是RSS，虽然现在RSS的资源真是越来越少了。
  4. 多账号管理，可以和朋友一块使用。自己搭建，邀请几个好友一块用。
  5. 其他，自行研究。

搭建KindleEar

搭建GAE平台

看了下官方的部署方式，过于麻烦，还好有民间有第三方自动部署脚本。不过由于GAE平台及脚本一直在变动，中间也遇上不少坑

KindleEar只能运行在Google的GAE（Google APP Engine），GAE是谷歌的一款产品，它提供了一个web程序的运行空间，而且GAE可是免费的，非常稳定。你需要自己先准备一个google账号，最好是申请一个新号，因为大部分老号已经打开了二步验证，无法开放以下安全设置。

首先，打开应用的安全设置

创建一个GAE项目

进入上述界面后，左上角点击选择项目
点击“+”号新建一个项目

输入项目的名字，可以随便填写。4~30 个字符，接受字母、数字、半角引号、连字符、空格、感叹号。

生成的ID记录下来

遇到的坑

1、死活打不开GOOGLE云端控制台，检查半天结果发现是CHROME插件的问题，关闭所有插件后解决。最后发现是LASTPASS的锅

2、KINDLEEAR首页我的订阅打不开，结果导入了我的OPML列表后显示出来，也就没去找原因了

3、GAE被强的厉害，即使用代理也经常无法正常访问，尝试在全局代理下才能稳定访问

4、可能是第3个原因，导致投递很慢，一次推送大概要2-10分钟左右

5、10M以下投递都成功了，20M的显示超时，但KINDLE上还是收到了推送，好像AMAZON限制是最高50M。所以把OPML进行了精简

6、采用的是脚本布置，之后一直没有自动推送，检查台cron为空。看到的解决方法都是基于手动部署的，于是干脆重新运行./uploader.sh。终于看到后台cron中终于有了/deliver作业，希望能运行

总结

KINDLEEAR效果完美，可随意添加RSS地址可导入OPML列表。缺点是GAE被强的厉害，没有SS就别想配置
http://wheremylife.cn：效果堪称完美，但自定义添加RSS很多都加不了，不过推荐的RSS也不错
Push to Kindle:最好的CHROME推送工具，可以完美支持图片。

Pocket 党
当自己订阅的 rss 源有新的更新时，自动保存到 Pocket。
ifttt.com/recipes/9733
将自己在 Pocket 里标注喜欢的文章，同步保存到印象笔记里。
ifttt.com/recipes/70594
http://p2k.co ：定時/定量推送pocket文章到kindle，很好用，有图片。可惜免费用户一次只能更新3条，且一天只能自动同步一次。会员太贵3美元一个月
POCKET没有好用的推送工具。

准备方案

将最常读的RSS用kindleear来定时推送，其它网文用Push to Kindle推
