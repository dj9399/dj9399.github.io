树莓派3安装多系统

发表于 2017-04-27

4/27

背景需求：现用OSMC，感觉把下载和视频播放都做在一起很不方便，因为俺分了二个硬盘（一个笔记本硬盘作为下载盘，供电低，一个是普通3.5硬盘作为仓库盘），却只有一根易驱线。这样每次看视频时都要：1、把自动下载服务取消 2、关机 3、换硬盘 4、开机。如果用berryboot多系统的话，步骤可改为改为：1、关机 2、换硬盘 3、开机切换系统。确实会简单很多，而且一系统一用会有更多方便之处。

工具：派、HDMI线、带HDMI的显示器、linux虚拟机或实体机(用来读写EXT4格式的U盘)

参考资料：

https://fnzoo.com/article/info/67476.html

需要提醒一下的地方：

WINDOWS下的EXT2FSD工具太不稳定了，我在WIN7系统上就遇上N次的EXT2FSD死掉，无法读取，只好反复重启还是无法解决

通过迅雷把需要安装的系统下载到EXT4的U盘中，再通过U盘安装时一定要按住“ADD OS”按钮不动才会弹出从U盘安装选项！

安装了Raspberry +openelec，berryboot自带系统中没有OSMC，只有用openelec了，不过看来也很不错

Raspberry 下的vi完全是反人类设计，立马改用nano

5/2

但经过周末的测试和使用，发现在从berryboot官网上下载的OPENELEC安装xbmc-addons-chinese插件完全正常，但是无法支持NTFS硬盘的读取，而且OPENELEC下是无法使用APT-GET命令来安装软件。于是又重新安装了一个OSMC FOR berryboot的最新系统，这次呢安装xbmc-addons-chinese插件后，里面的插件全部无法加载…而且无法读取NTFS硬盘，#安装所需软件包apt-get install fuse-utils ntfs-3g #加载内核模块modprobe fuse直接报错，然后网上乱查询一通报错–基本无法解决，具体就不记录了，总之就是一个坑

5/18

berryboot已经用了半个月，陆续又安装了几个比较新的系统。然并卵，各个系统用得正常，但只要更新或安装软件，一会报错无依赖，一会报错版本太低的。网上一通搜索确认是坑后，无耐之下，干脆把16G的SD卡改装NOOBS。毕竟NOOBS是树莓官方指定的多系统工具

安装NOOBS的过程很简单，网上教程也一大把，就不详述。我是通过NOOBS安装了Raspberry 、libreelec、OSMC三个系统

NOOBS安装中的坑：
1、FAT32无法分区错误 “Error resizing existing FAT partition”
解决办法： SDFormatter设置允许Logic partition 扩展 ON，重新格式化一次
2、官方下载的完全安装包只包含Raspberry 、libreelec，其它系统需要从网上下载。我尝试从网上安装OSMC，必须要FQ，否则会报错下不了

5/22

安装之后的感受:

1、所有的系统都和原版一样，安装更新软件顺畅。还是官方靠谱啊，不过缺点就是不能像berryboot一样使用方便（可以随意增删系统）

2、以前一直觉得OSMC系统好，现在用了Raspberry 推荐的libreelec才发现，官方推荐的才是靠谱。libreelec包含了OSMC系统的所有优点（都是KODI方案）还修正了BUG，内置自动挂载U盘，自带支持NTFS、EXFS，自带字幕的影片也可以显示字幕。唯一的缺点就是不带apt-get，只能通过add-on插件方式来安装系统软件

6/11

noob下的Raspberry 才使用了半个月，就挂了（每天15小时左右下载）。看来还是得做专系统专用的隐定…于是慢慢的改为4G卡安装libreelec，8G卡安装KALI，16G卡安装Raspberry，从此脱坑
