树莓派3配置kali

发表于 2017-06-12

安装过程就不写了，简单而且教程一搜索大把
奇怪的是kali连接名称竟然是“recovery”，我还是根据MAC地址才找到了它
还好重启后正常了，变回”kali”

一、设置为国内的更新源

编辑更新源所在文件：nano /etc/apt/sources.list

##阿里云kali源
deb http://mirrors.aliyun.com/kali kali-rolling main non-free contrib
deb-src http://mirrors.aliyun.com/kali kali-rolling main non-free contrib
deb http://mirrors.aliyun.com/kali-security kali-rolling/updates main contrib non-free
deb-src http://mirrors.aliyun.com/kali-security kali-rolling/updates main contrib non-free

##中科大kali源
deb http://mirrors.ustc.edu.cn/kali kali-rolling main non-free contrib
deb-src http://mirrors.ustc.edu.cn/kali kali-rolling main non-free contrib
deb http://mirrors.ustc.edu.cn/kali-security kali-current/updates main contrib non-free
deb-src http://mirrors.ustc.edu.cn/kali-security kali-current/updates main contrib non-free

二、安装更新软件

刷新软件列表：
apt-get update
安装net-tools(没有ifconfig) ：
apt-get install net-tools
安装中文字体 ：
apt-get install ttf-wqy-zenhei
输入
dpkg-reconfigure locales
把下面的三个选择上（按空格键选择）其余取消选择
en_US.UTF-8、zh_CN.GBK、zh_CN.UTF-8

为了减少麻烦，把所有的组件全部装上 ：apt-get install kali-linux-all。此步慎重,下载加安装花了近四个小时的时间，一共占用了8G左右的内存，非16G以上卡不要这一步，安装完成后最好重启一下树莓派。reboot

三、安装远程:

本来安装VNCSERVER，不过界面一直灰屏，于是改装XRDP

apt-get install xrdp
/etc/init.d/xrdp start

之后直接使用WINDOW远程
