proxmox(pve) LEDE OMV笔记

发表于 2020-04-03

需求：一直想弄台PVE，暂时只需要LEDE旁路由，OMV做个NAS

硬件：从附近收到台还行的小主机，二代I3，关键H61主板自带千兆网口，30G SSD的，4G DDR3，自己再加根4G，还不错，就是机箱大了一点。这配置来跑PVE是非常合适，功耗低，性能够用，比起J1900 ARM板强多了，日后还可以升到E3或I7CPU，升级空间巨大

其它说明：NAS是利用手上几块老硬盘，随时可能会挂，就不需要做RAID、ZFS这些高深的东西了。单盘共享，做好备份，做好随时挂的准备

感想：PVE果然NB，安装简单，适用性强，这二点比esxi强太多。但稳定性还是不及esxi，强烈推荐家用及测试用PVE

PVE：https://192.168.1.127:8006
pve基于Debian,做虚拟机最少需要2个CPU，2G内存（开机占1G），建议至少4CPU，4G内存
上传ISO目录:/var/lib/vz/template/iso

LEDE（VMID:101）:1G硬盘足够 512内存 ip:192.168.1.124

OMV（102）：ip：192.168.1.158
系统安装会独占一整块盘，安装时一定要接入网络 最低系统1.6G,内存消耗160M
实际使用推荐内存给2G，硬盘8G，硬盘分4G都不够用造成插件无法更新

win7（103）：ip：192.168.1.166

pve安装配置

安装

安装倒是轻车熟路，不过在选择安装目录的时候，一定要分配全部磁盘空间，否则较难使用未创建分区的空间（需要Shell>fdisk），造成浪费。之后的实测已确认虚拟机磁盘是可以存放在PVE的系统盘上的。

配置

关闭订阅提醒
打开（使用VIM或者nano）编辑/usr/share/javascript/proxmox-widget-toolkit/proxmoxlib.js
将if(data.status!==’Active’){替换为if(false){
然后保存，重启网页服务
systemctl restart pveproxy
然后CTRL+F5强制刷新网页，或清理一下浏览器的缓存就好了！

更新Debian库
请删除/etc/apt/sources.list.d/pve-enterprise.list，并修改/etc/apt/sources.list.d/pve-install-repo.list，添加一行deb http://download.proxmox.com/debian/pve stretch pve-no-subscription，保存即可。

添加社区源

echo “deb http://download.proxmox.wiki/debian/pve stretch pve-no-subscription” >/etc/apt/sources.list.d/pve-install-repo.list

目前Proxmox中文社区源已正式提供镜像服务，镜像更新频率为每10小时，镜像站机房存放在法国巴黎online机房使用阿里云、华为云融合提供国内CDN加速服务。
地址
http://download.proxmox.wiki

删除企业源
rm -rf /etc/apt/sources.list.d/pve-enterprise.list

下载秘钥
wget http://download.proxmox.com/debian/proxmox-ve-release-5.x.gpg -O /etc/apt/trusted.gpg.d/proxmox-ve-release-5.x.gpg

建议同时使用国内debian源
vim /etc/apt/sources.list
deb http://mirrors.aliyun.com/debian/ stretch main non-free contrib
deb-src http://mirrors.aliyun.com/debian/ stretch main non-free contrib
deb http://mirrors.aliyun.com/debian-security stretch/updates main
deb-src http://mirrors.aliyun.com/debian-security stretch/updates main
deb http://mirrors.aliyun.com/debian/ stretch-updates main non-free contrib
deb-src http://mirrors.aliyun.com/debian/ stretch-updates main non-free contrib
deb http://mirrors.aliyun.com/debian/ stretch-backports main non-free contrib
deb-src http://mirrors.aliyun.com/debian/ stretch-backports main non-free contrib
最后执行

apt update
apt update&&apt dist-upgrade #升级pve

lede安装配置

参考：从零开始组建一台NAS+软路由（2）–https://lala.im/5433.html
LEDE的固件：
http://firmware.koolshare.cn/LEDE_X64_fw867/虚拟机转盘或PE下写盘专用/
下文件名含有combined-squashfs的即可，我这边就直接登录到PVE内使用wget下载：
wget http://firmware.koolshare.cn/LEDE_X64_fw867/%E8%99%9A%E6%8B%9F%E6%9C%BA%E8%BD%AC%E7%9B%98%E6%88%96PE%E4%B8%8B%E5%86%99%E7%9B%98%E4%B8%93%E7%94%A8/openwrt-koolshare-mod-v2.31-r10822-50aa0525d1-x86-64-combined-squashfs.img.gz
将固件解压出来：
gzip -d openwrt-koolshare-mod-v2.31-r10822-50aa0525d1-x86-64-combined-squashfs.img.gz
转换格式为KVM支持的qcow2:
qemu-img convert -f raw -O qcow2 openwrt-koolshare-mod-v2.31-r10822-50aa0525d1-x86-64-combined-squashfs.img vm-101-disk-0.qcow2
注：vm-101-disk-0.qcow2，其中的101就是一开头让你记住的VMID号，而后面的disk-0，这里的0指的是这台虚拟机的第一块磁盘。因为我们之前已经把0号磁盘删了，所以我们再手动创建的磁盘这里肯定是0号。
最后将转换完成的磁盘镜像导入到之前创建的虚拟机：
qm importdisk 101 vm-101-disk-0.qcow2 local-lvm
注：这里的101同理也是指的VMID号，而后面的lock-lvm指的是你要将这个磁盘镜像存放在哪里，这个可以在PVE的WEB界面查看的到：
至于科学上网的部分，已经在插件中被和谐了，动手网上搜索下载，手动上传即可解决

直通硬盘

直通前先把102（要挂载的虚拟机ID）关闭
查看硬盘id通过ls /dev/disk/by-id(lsblk查看所有硬盘设备)
缓存盘：
ata-ST2000VM003-1CT164_W1H37R9K sata1 sda QM00007
ata-ST2000VM003-1CT164_W1H3600D sata2 sdb QM00009

qm set -sata2 /dev/disk/by-id/硬盘id
qm set 102 -sata1 /dev/disk/by-id/ata-ST2000VM003-1CT164_W1H37R9K
qm set 102 -sata2 /dev/disk/by-id/ata-ST2000VM003-1CT164_W1H3600D
qm set 102 -sata3 /dev/disk/by-id/ata-ST2000NC001-1DY164_Z1E3MBXQ
如果返回以下信息,说明已成功挂载
update VM 102: -sata1 /dev/disk/by-id/ata-WDC_WDXXXX_XXXX_XXXX

问题

1、pve界面omv无法关机
Proxmox VE 安裝 VM上的 Qemu-guest-agent
使用 VM 時需要使用 Agent 才可從系統去跟 VM 溝通

PVE主机，客户机”选项”中开启”Qemu代理”
客户机linux版本安裝方式
apt-get install qemu-guest-agent -y
之后必需重启PVE主机才能生效

PVE主机中进行Testing：
qm agent ping

2.win7无法访问samba服务器
https://blog.csdn.net/lizongyao/article/details/89054634

3.按2配置之后，WIN7继续报错“未知的用户名……”
修改配置文件”/etc/samba/smb.conf”中的”log level = 3”,然后观察日志”tail -f /var/log/syslog”,
观察到如下日志输出主要报错是：ntlm_password_check: NEITHER LanMan nor NT password supplied for user
通过在smb.conf中进行设置来接受不支持的协议：
vim /etc/samba/smb.conf
在[global]最后加入：
lanman auth = yes
ntlm auth = yes
重启后解决（or：/etc/init.d/samba-ad-dc restart）

4.按方法3配置，每次OMV修改配置都会覆盖配置文件，导致每次都要手动修改文件
查询是WIN7默认关闭了SMB1和SMB2，手动开启
win7开启SMB1\2:https://chuill.com/post/112.html
最后在家中电脑上正式安装时发现，只要开启SMB1/SMB2，问题2、3即可解决

win7直通显核

顺手就找个精简老毛子版本来装个WIN7试试直通
参考：https://lala.im/6324.html
花了半天时间配置，安装驱动后重启就蓝屏，先不试，回滚后恢复正常。

lspci -nn | grep VGA
00:02.0 VGA compatible controller [0300]: Intel Corporation Xeon E3-1200 v3/4th Gen Core Processor Integrated Graphics Controller [8086:0412] (rev 06)

lspci -vvv -s 00:02.0

echo “options vfio-pci ids=8086:0412” >> /etc/modprobe.d/vfio.conf

打开文件 /etc/pve/qemu-server/103.conf（我的Win10在PVE顺序是103），添加这两行内容后保存：
args: -device vfio-pci,host=00:02.0,addr=0x18,x-igd-opregion=on
vga: std
