drupal8安装之php入坑

发表于 2017-11-13

公司需要在redhat6.4下安装drupal8，用于测试，从网上搜索教程一路上遇上不少坑，原因还是LINUX太灵活，大家的需求稍有变动，安装过程立马变样。因此把安装过程记录一下
原系统php没有安装fpm-php,使用yum安装，或者重新下载编译安装均报错，想重装php-5.6.23,php官网又只有php-5.6.32，只好干脆重装php。
使用 yum 对 php 进行删除，如：yum remove php yum remove php-common
当然 yum 是很大可能删不彻底的，现需要用 rpm 命令进行删除，命令：rpm -qa|grep php，如果系统还存在有php该命令就会显示出来，格式大致如下：

php-pdo-5.1.6-27.el5_5.3

php-MySQL-5.1.6-27.el5_5.3

php-xml-5.1.6-27.el5_5.3

php-cli-5.1.6-27.el5_5.3

php-common-5.1.6-27.el5_5.3

php-gd-5.1.6-27.el5_5.3

然后再使用rpm -e 命令逐个删除，如：rpm -e php-pdo-5.1.6-27.el5_5.3
如果使用 php -v 查到php还存在，需要强行删除。首先执行：php -info 查看 php 相关环境，包括扩展库的位置，php 程序所在位置等，根据提示删除相应的目录就可以了，我在执行php -info 后显示的库路径为：/usr/local/lib/php/extensions,程序路径为：/usr/local/bin/php，我是将/usr/local/bin/php目录整个删除的。
至此，php 在 centos 基本是删除成功了，再使用 php -v 等函数就会失败，表明php删除成功了。

安装编译新PHP
./configure –prefix=/usr/local/php \
–enable-fpm \
–enable-opcache \
–enable-mysqlnd \
–enable-zip \
–enable-pdo –with-pdo-mysql –with-pdo-sqlite \
–enable-mbstring \
–enable-exif \
–with-fpm-user=nginx \
–with-fpm-group=nginx \
–with-openssl \
–with-mysql \
–with-mysqli \
–with-curl \
–with-zlib \
–with-gd \
–with-mcrypt

make && make install

编译时提示
configure: error: mcrypt.h not found. Please reinstall libmcrypt
需要安装libmcrypt
1、安装第三方yum源
wget http://www.atomicorp.com/installers/atomic
sh ./atomic
2、使用yum命令安装
yum install php-mcrypt libmcrypt libmcrypt-devel

安装完毕，下面开始配置php
cp php.ini-production /usr/local/php/php/php.ini
cp /usr/local/php/etc/php-fpm.conf.default /usr/local/php/etc/php-fpm.conf
编辑 php.ini
vi /usr/local/php/php/php.ini
查找 cgi.fix_pathinfo= 并将其修改为如下：
cgi.fix_pathinfo = 0
查找 date.timezone 修改为：
date.timezone = Asia/Shanghai
编辑 php-fpm.conf
1
vi /usr/local/php/etc/php-fpm.conf
查找并配置以下内容：

user = nginx
group = nginx

listen.owner = nginx
listen.group = nginx

listen = /var/run/php-fpm/php-fpm.sock
这里使用的是 socket 方式连接。
可能需要先创建 php-fpm 文件夹
mkdir /var/run/php-fpm/
接下来配置 php-pfm 服务：
cp sapi/fpm/init.d.php-fpm /etc/init.d/php-fpm
chmod +x /etc/init.d/php-fpm

chkconfig –add php-fpm
chkconfig php-fpm on
启动 php-fpm：
service php-fpm start

然后在配置 Nginx 后端支持的时候：
location ~ .php$ {
try_files $uri =404;
fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
include fastcgi_params;
fastcgi_pass unix:/var/run/php-fpm/php-fpm.sock;
}

至此PHP环境安装好，测试可以网上搜索下
